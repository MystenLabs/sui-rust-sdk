Expand description

Fold a sequence of batches into head, returning the updated head.

For each batch in order, this:

  1. Computes the merkle root over the batch’s commitments via build_event_merkle_root.
  2. Reinterprets the 32-byte root as a U256 in little-endian.
  3. Folds the result into head.mmr using BLAKE2b-256 carry-propagation: while the lowest empty slot is occupied, hash that peak with the carry and try the next slot up; otherwise drop the carry into the slot. If the carry propagates past the highest occupied level, push a new slot.
  4. Advances head.checkpoint_seq to the batch’s checkpoint and increments head.num_events by the number of commitments folded.

The MMR fold matches the on-chain Move implementation byte-for-byte — see test_mmr_digest_compat_with_rust under sui-framework/sources/accumulator_settlement.move. Any divergence here breaks reconciliation against fetched EventStreamHead objects.
